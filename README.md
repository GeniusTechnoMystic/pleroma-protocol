# Pleroma Protocol

[ˈplɛrəmə | from Greek πλήρωμα — *fullness, the totality of powers and knowledge*]

> **A decentralized meta-network for federating, stewarding, and trust-weighting all human knowledge — across languages, sources, knowledge graphs, and knowledge systems — with rigorous epistemic controls and graded access aligned with the long-term flourishing of all life.**

---

## Why

Human knowledge is fragmented across languages, paywalls, academic silos, and competing knowledge graphs. Wikidata can't talk to OriginTrail. OpenAlex can't talk to museum archives. There is no shared trust framework, no graded access model, no protocol for AI agents to attest their intentions before requesting knowledge.

Pleroma Protocol is that missing layer.

## What It Is

Pleroma is **not** a new knowledge graph. It is a **meta-network layer** — the TCP/IP for the Internet of Knowledge Graphs:

| Layer | What it does |
|-------|-------------|
| **Federation Protocol** | Cross-graph discovery, query routing, trust metadata exchange between independent KGs |
| **Epistemic Overlay** | Bayesian trust scoring, provenance chains, epistemic mode tags across source boundaries |
| **Graded Access Middleware** | Tier 0–4 access control gating query results based on agent attestation |
| **Agent Attestation Protocol** | Cryptographic identity + operational ontology + telos disclosure for AI agents |
| **DAO Governance Layer** | Cross-system governance — Classification Council, Ontology Guild, Ethics Panel |
| **Steward Agent** | An autonomous AI agent monitoring and optimizing knowledge flow across the entire federated network |

## What It Integrates

Pleroma peers with existing systems rather than replacing them: **Wikidata**, **OriginTrail DKG**, **OpenAlex**, **Solid/Inrupt**, **KBpedia**, **OCLC/WorldCat**, **Europeana**, **Internet Archive**, **IPFS/Filecoin**, **Commons Stack / Aragon** DAO tooling, and more.
